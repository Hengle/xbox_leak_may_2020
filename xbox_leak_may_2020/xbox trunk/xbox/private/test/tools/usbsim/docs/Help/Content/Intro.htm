What is a USB Simulator
The purpose of the USB Simulator is to dynamically imitate any given USB device in real time. A separate computer controls the "virtual" devices, so the simulator acts like a proxy by transferring USB commands and responses between the host and the secondary computer.
Uses
  •  Test the host USB stack
    • Different types of devices: Interrupt, Bulk, Isocronous
    • Stress IO, device enumeration, etc.
    • Bad responses
  •  Testing host applications by automating user input
  •  Quick prototyping of new USB devices
Features
  •  Up to four USB devices on each simulator
  •  Supports bulk / interrupt and Isocronous transfers
  •  Access to the electrical USB signals for probing
  •  Auto send feature allows prompt responses
