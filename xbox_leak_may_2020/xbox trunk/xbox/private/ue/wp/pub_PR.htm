Overview: How to Work with Xbox PR

Press releases and other publicity related to third parties, and to the Xbox™ video game console from Microsoft, should occur only when a signed Publisher License Agreement (PLA) exists between Microsoft and the third-party partner. Until a contract is signed, no publicity can be conducted for a product related to Xbox. However, once the PLA is signed, Xbox PR will encourage its third-party partners to let the media know what is exciting and unique about its Xbox-related product.

Artwork

Xbox PR requires that all screen shots distributed to the media on your game be taken on the Alpha 2 XDK (Xbox Development Kit) or subsequent versions of the XDK. The goal here is to only have screen shots in circulation that represent what the console is capable of producing, rather than screen shots taken from early development kits that produce inferior results. The Alpha 2 XDKs are being delivered to developers and publishers in the December 2000 and January 2001 time frame. Any artwork that is given to the media that is not actually a screen shot should be correctly identified as such. Rendered images can be created using any medium and sent to the media as a visual portrayal of the game, but it must be clearly stated that these are not actual screenshots.

Press Releases

All press releases mentioning the Xbox video game console must undergo a mandatory review and be approved by Xbox PR. See the attached Press Release Approval Process sheet for more details. Here are some additional suggestions/guidelines for your releases:

  • In the case of peripheral and middleware partners, tailor the release to explain how your product will make Xbox perform better than any other console on the market.


  • In the case of game software publishers, explain how the technologic innovations in Xbox make your game do things that were never before thought possible. Outline in your release what it is about Xbox that brings your game to life and creates such a rewarding experience for the user.

Tracking Program

Many of our third-party partners have said that they would appreciate having a central coordinator at Xbox PR that keeps a master schedule of upcoming events and press releases pertaining to Xbox.

  • Participation in this tracking program, whereby Microsoft would track your upcoming press releases and events, is not mandatory. However, it is encouraged so that we can avoid having a logjam of releases and/or events on the same day.

  • Our goal is to coordinate the PR for all Xbox titles and to recommend the best days to hold events, distribute news, and release product information to maximize coverage. This helps promote both the individual third-party games and the overall platform.

Press Release Approval Process

Any press release discussing the Xbox video game console must be submitted to Xbox PR for approval.

Steps for Approval

  1. Write release.


  2. Check to see that Xbox is referred to correctly in the release by reviewing the Style/Trademark Guidelines in the following section.


  3. Submit release for approval. E-mail it to xboxpr@xbox.com at least two business days before desired distribution date and time.


  4. Call a member of the Xbox PR team (see contact list below) to alert us that the release has been e-mailed. We will be checking our e-mail regularly, but it is possible that an e-mail could not get through and we would not see it, in which case it would need to be resent.


  5. Representatives from the Xbox PR team review the press release and make any edits using Microsoft® Word, with revision marks on.


  6. Xbox PR e-mails press release back to third party within two business days of receiving the original document.


  7. Third-party partner incorporates edits made by Xbox PR team.


  8. If the third-party partner wants to resubmit the release to Xbox PR with additional edits, add another business day into the timeline to account for this additional editing.


  9. Final version of release must be sent back to xboxpr@xbox.com before distribution.

Style/Trademark Guidelines

Official Descriptor

Use the official descriptor the first time Xbox is mentioned:

  • the Xbox™ video game console from Microsoft

    • Note   Xbox trademarking is not needed in titles of press releases, just in the first mention of Xbox within the body of the release.

How to Refer to Xbox

  • Please note the correct spelling of Xbox-a capital "X" followed by lower case "box," and it is all one word.


  • After first describing Xbox in the release as "the Xbox™ video game console from Microsoft," it is acceptable to refer to the console as "Xbox."


  • Avoid using Microsoft as a qualifier directly before the word Xbox, such as calling the console "Microsoft's Xbox."


  • Avoid using Xbox in the possessive, such as "Xbox's technical specifications." Instead, say, "the technical specifications of Xbox."


  • Avoid using Xbox in the plural, such as "in the booth will be numerous Xboxes." Instead, say that, "in the booth will be numerous Xbox video game consoles."

Microsoft's Legalese

  • Xbox and Microsoft are trademarks of Microsoft Corp.

How to Take a Screen Shot on an Alpha 2 XDK

Screen Shots for Xbox Games

The Xbox system software will not support a system wide screen shot solution. We have provided an API that will allow developers to implement their own screen shot functionality in their title. To do this, you will need to add some code to your title as described below.

Xbox Programmatic Screen Captures Under Xbox System Software

One method of producing high-fidelity screen captures for Xbox games running under the Xbox system software is to incorporate the below code, and then copy the file onto a machine that will allow for file type/bit depth conversion. One can also use the below code as a basis for dumping a surface to another format, with modification of course. Questions or comments can be e-mailed to xboxds@xbox.com. 

//-----------------------------------------------------------------------------
// Name: XBUtil_DumpSurface()
// Desc: Writes the contents of a surface (32-bit only) to a .tga file. This
// can be a backbuffer, texture, or any other 32-bit surface.
//-----------------------------------------------------------------------------
HRESULT XBUtil_DumpSurface( LPDIRECT3DSURFACE8 pSurface, CHAR* strFileName )
{
    // Get the surface description. Make sure it's a 32-bit format.
    D3DSURFACE_DESC desc;
    pSurface->GetDesc( &desc );
    if( desc.Size != ( desc.Width * desc.Height * sizeof(DWORD) ) )
        return E_NOTIMPL;

    // Lock the surface.
    D3DLOCKED_RECT lock;
    if( FAILED( pSurface->LockRect( &lock, 0, 0 ) ) )
        return E_FAIL;

    // Setup the TGA file header.
    struct TargaHeader
    {
        BYTE IDLength;
        BYTE ColormapType;
        BYTE ImageType;
        BYTE ColormapSpecification[5];
        WORD XOrigin;
        WORD YOrigin;
        WORD ImageWidth;
        WORD ImageHeight;
        BYTE PixelDepth;
        BYTE ImageDescriptor;
    } tgaHeader;

    ZeroMemory( &tgaHeader, sizeof(tgaHeader) );
    tgaHeader.IDLength        = 0;
    tgaHeader.ImageType       = 2;
    tgaHeader.ImageWidth      = (WORD)desc.Width;
    tgaHeader.ImageHeight     = (WORD)desc.Height;
    tgaHeader.PixelDepth      = 32;
    tgaHeader.ImageDescriptor = 0x28;

    // Create a new file.
    FILE* file = fopen( strFileName, "wb" );
        if( NULL == file )
        {
            pSurface->UnlockRect();
                return E_FAIL;
	}

    // Write the Targa header and the surface pixels to the file.
       fwrite( &tgaHeader, sizeof(TargaHeader), 1, file );
       fwrite( lock.pBits, sizeof(BYTE), desc.Size, file );
       fclose( file );

    // Cleanup and return.
    pSurface->UnlockRect();

    return S_OK;
}
