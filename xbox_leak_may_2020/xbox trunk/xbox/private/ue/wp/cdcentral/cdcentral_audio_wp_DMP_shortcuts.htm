Keyboard Shortcuts for DirectMusic Producer

By Scott Selfon, Audio Content Consultant, Content and Design Team, Xbox Advanced Technology Group

Microsoft Confidential

This is part of a series of "column-style" white papers on various aspects of Microsoft® DirectMusic® Producer, the content creation tool for DirectMusic. If you would like to see a specific area addressed, please send e-mail to content@xbox.com.

Introduction

Microsoft® DirectMusic® Producer, the content authoring tool for the Xbox™ video game system from Microsoft, has been given the enormous task of exposing every feature of DirectMusic, from low level note variability to full blown segment editing, from Microsoft® DirectX® Audio scripting to Downloadable Sounds (DLS) collection manipulation. Suffice it to say, there are quite a few tools that DirectMusic Producer has been tasked with exposing to the user in a way that gives more control to content authors. Often, these tools can appear unfamiliar or difficult to use, because they have no linear equivalents; sequencers and wave editing software don't generally expose the functionality of making their content interactive, variable, or dynamic. This white paper will be a "cheat sheet" of DirectMusic Producer shortcuts that simplify the process of creating content.

MIDI and Wave Clipboards

First and foremost, DirectMusic Producer is not meant to replace your existing wave editing and sequencing software. DirectMusic Producer provides the most basic sequencing and wave editing functionality, but advanced manipulation is generally left to more specialized applications. There are many ways to use these applications in tandem, from using DirectMusic Producer simply as a DLS file manager1 (and method of communicating to the Microsoft Software Synthesizer2) with a sequencer triggering the samples, to full integration, where content authored in other applications is imported into DirectMusic Producer, either via wave/MIDI import options, or via the clipboard.

Copying content via the clipboard is a fairly straightforward process. For wave data, select the waveform in your wave editing software and use the copy command (generally CTRL+C). Then in DirectMusic Producer, either open an existing wave or create a new wave (making sure it's the same sampling rate; DirectMusic Producer does not convert sample rates), open the wave editor window, and use the paste command.

For sequenced data, a single track can be added the exact same way. Use the copy command to take the information from your source sequencer, click the mouse into the sequence track, pattern track, or pattern part (in a style), and add it using the paste command.

To copy multiple tracks, use of the rectangular track selection button to the left of each part you wish to copy information from or to:

  1. Select one or more tracks in your sequencer. Then in DirectMusic Producer, click the track selection button next to each performance channel you wish to add tracks to. The button will turn red (or orange, if the track happens to be the currently active audition track3).



    Illustration  The rectangular track selection button. A track selected to copy multiple tracks from will be red (or orange, if it's the current audition track); an unselected track will be gray (or yellow, if it's the current audition track).

  2. Once all of the buttons are selected, click the position in the timeline where you want the information to be added. If you only want to add tracks to a specific area (that is, not the entire length of the segment), drag select it from the timeline.



    Illustration  Click in the timeline to determine where the information should be added. Timeline clicks "snap" to the setting you choose from the right click menu (None, Grid, Beat, or Barline).

  3. Now use the paste command (CTRL+V defaults to paste merge, or you can click Edit, click Paste, then choose either Merge or Overwrite from the menu). Each track that was taken from your sequencer is added to the corresponding track of your segment or pattern (in a style). Note that performance channels need not correspond, so if you took information from MIDI tracks 1 to 4 of your sequencer, they need not be added to pchannels 1 to 4.
