Best Practices for Xbox Audio Design:
Chapter 2 - Xbox Audio Content

By Scott Selfon, Audio Content Consultant, Xbox Content and Design Team

Microsoft Confidential

Introduction

This document will discuss the formats and kinds of audio content that are supported on the Xbox™ video game system from Microsoft. As discussed in chapter 1, the Xbox implements a platform-specific version of the Microsoft® DirectX® 8.0 libraries. Many of the file formats and content specifics supported by the Microsoft® Windows® APIs are supported on Xbox, even while the interfaces to play that content may be modified. If you have questions about content compatibility and authoring, or requests for content support not detailed below, please contact us at content@xbox.com.

Note   This document is not a complete specification of the Microsoft® DirectSound® and Microsoft® DirectMusic® file formats, but rather a list of the content formats supported. For more detailed information, please consult the Xbox Development Kit (XDK) documentation.

DirectSound (Wave) Content for Xbox

The Xbox Media Communications Processor (MCP) is a powerful hardware synthesis engine, with advanced audio synthesis and processing capabilities. The goal is to give audio designers much more flexibility when authoring a variety of content while still minimizing disk and memory footprint for the running application. The presence of the Xbox MCP means that many scenarios that would require multiple versions of content to be authored on previous platforms can now use the same source media to accomplish these effects. For instance, the real-time hardware occlusion and obstruction filtering means a single source wave can be used when a sound originates right in front of the player as well as when it comes from behind a door or wall.

Comparisons with DirectSound on the Windows Platform

Wave files continue to be the preferred format used for DirectSound playback of audio data. However, there are a few differences to note:

  • The DirectSound libraries for Xbox system software natively support adaptive differential pulse code modulation (ADPCM) and Microsoft® Windows Media™ Audio (WMA) compression only. Unlike the Windows platform, Xbox system software will not have an Audio Compression Manager (ACM). This means that compression schemes other than those listed below will require the authoring and delivery of an Xbox Media Object along with the title, which will decode the audio data in software.


  • Xbox supports multichannel source audio. You should use linear uncompressed pulse code modulation (PCM) WaveFormatEx wave files for this.

Wave Formats Supported by Xbox

The Xbox supports 8- and 16-bit PCM data, 4-bit Xbox ADPCM data (proprietary format), and WMA-compressed wave data. An Xbox-specific implementation of WAVEFORMATEX is used to specify the number of channels. Up to six interleaved channels of data may be used. The following wave formats and specifications are supported:

Specification Supported Formats Comments
Wave format (uncompressed) Linear PCM
Wave format (compressed) ADPCM, WMA For DLS-2, ADPCM support
Channels Mono, stereo, 4.0, 5.1 (up to six discrete channels) Dolby® Digital ("AC-3") encoded source waves not supported as source content within Xbox MCP
Sampling rate 188 Hz to 48 kHz Upsampled in hardware to 48 kHz (multipoint interpolation)
Bits per sample 8 bits, 16 bits (PCM/WMA)
4 bits (ADPCM)		 Xbox MCP uses 24-bit resolution internally, outputs 20-bit

Unsupported Wave Formats

The Xbox MCP will not support some formats:

Specification Unsupported Formats Comments
Wave format (uncompressed) Pre-encoded formats, such as Dolby Digital and DTS * Use 6- or 4-channel PCM for surround
Wave format (compressed) MPEG-1 Layer 3 (MP3), Windows-supported codecs Use Xbox Media Objects for custom compression scheme decode (in software)
Channels 3 channels, 5 channels, >6 channels Use 6 or 4 channel waves (hardware can automatically mix down to stereo)

*  Note that an application is free to bypass the Xbox MCP entirely and write directly to the analog or digital outputs, which would allow for such custom-encoded formats. Be aware that this would preclude the use of hardware mixing or effects.

Prerecorded Audio Recommendations

For prerecorded musical scores and ambience, we recommend delivering two versions of audio:

  1. 48-kHz PCM wave with six discrete channels (for 5.1 playback)


  2. 48-kHz PCM wave with two discrete channels (for stereo playback), optionally Dolby® Surround encoded

If you want to deliver prerecorded audio in compressed form, remember that each hardware buffer can be routed to up to eight mix bins, including the six specific output channels, as you see fit. For instance, you could create three stereo ADPCM streams (which would be six discrete channels of audio) and map them to 5.1 channels in hardware. This would not be recommended for WMA, because the software decode can spike to 10 to 15 percent of the CPU per WMA voice. Also, watch for bandwidth issues if you don't position the audio streams close to each other on the DVD or hard disk.

Note that if you wish to deliver only one version of audio, the six-channel mix can be mixed down within the hardware for playback on stereo and Dolby Surround systems. However, keep in mind that certain information may be altered or lost:

  • Existing LFE channel is ignored


  • Discrete left and right surround channels are merged into single mono surround


  • Nothing will be sent to the center channel unless it is explicitly addressed to the center

For real-time, 3-D-positioned audio, mono waves are required.

For more information on multichannel authoring and implementation, see the white paper entitled The Surround Experience: Authoring for and Implementing Multichannel Audio Playback



Looping Waves

DirectSound supports a single pair of loop points within a wave, so the entire buffer does not need to be looped. Looping on "one-shot" (in memory) waves is performed in hardware. Four pieces of information must be supplied:

  • Sample start


  • Loop start


  • Loop end


  • Sample end

These pieces of information are also what to specify for DLS-2 waves (if you're using DirectMusic).

Glitch-free looping is guaranteed for PCM waves on any boundary and ADPCM waves on 64 sample boundaries.

DirectMusic Content for Xbox

Though we have attempted to make the programming interface feature set leaner than the Windows APIs, the actual content support will remain mostly intact. Some features (detailed below) have been omitted from launch because they depend on libraries not supported at launch (see chapter 1 of this document).

DirectMusic Behavior for Xbox System Software v1.0

Audio designers can author DirectMusic content on their development machine, running Windows 2000 with both DirectX 8 and DirectMusic Producer 8 installed. The run-time files created with DirectMusic Producer can then be copied to the Xbox machine, where the content can be played by the DirectMusic libraries.

The following DirectX 8 features are not supported for Xbox System Software v1.0 (August), but will be added in a future Xbox System Software release:

  • Effects control using parameter control tracks (effects can be managed via code by developer)


  • WMA compression of waves in wave tracks


  • Arbitrary audiopaths (only standard audiopaths will be supported; effects can be authored by the developer)
  • 5.1 audiopaths (content can still be manually panned to six-channel output)

Attempting to play content using these features may result in an error.

Unsupported DirectMusic Content Formats

Xbox will not support the following features at launch:

  • DirectX Media Objects (DMOs)—supplanted by hardware effects in the Xbox MCP and Xbox Media Objects. Your developer can author a GP program that uses these effects and route DirectMusic content to such effects paths. See the section Authoring for Hardware DSP Effects (Global Processor of Xbox MCP) for more details.


  • DirectX Audio Scripting using VBScript (Xbox will support AudioVBScript only)—VBScript depends on oleaut32.dll, vbscript.dll, and several other files (~1 MB footprint) not native to Xbox, so it will not be supported.

Authoring for Hardware DSP Effects (Global Processor of Xbox MCP)

The Xbox MCP currently natively supports the following effects:

  • Amplitude modulation (mono/stereo)


  • Chorus (mono/stereo)


  • Crosstalk


  • Distortion


  • Echo (mono/stereo)


  • Flanger (mono/stereo)


  • I3DL2 reverb


  • IIR2 (Band Pass/Parametric EQ*)


  • "Mini" reverb


  • Mixer

*  Parametric EQ is also supported on a voice-by-voice basis in the Voice Processor (outside of the Global Processor [GP]). This is discussed in more detail in chapter 3 of the Best Practices for Xbox Audio Design document.

GP DSP programs can be authored by a developer or content creator using the XGPImage tool supplied with the XDK. Real-time changes to effect parameters and programs are performed programmatically by the developer. To use GP programs with DirectMusic content, the developer can obtain the buffer that a (standard) audiopath is mixing into and assign it to a particular effect chain.

Note that the parameters for these hardware effects won't always correspond to (software) PC DirectX Media Objects, and the qualities of the hardware effects are different from the software PC DMOs. You'll generally want to experiment with them on the actual Xbox hardware. Several sample applications provided with the XDK should assist you with this.

DirectMusic Producer Demo Content

In a subdirectory of DirectMusic Producer on the XDK CD (and also available at http://msdn.microsoft.com), there is a self-extracting executable containing the Demo8 project. This project highlights some of the many features of DirectMusic and how they are exposed in Producer. Because DirectMusic Producer is used to author content for both PC and Xbox titles, some of the content in Demo8 is not applicable to the Xbox. In particular:

  • Much of the content uses instruments from the General MIDI DLS collection (GM.dls), which is delivered with Windows operating systems. This content will not work on Xbox without first authoring custom DLS instruments.

In addition, the author should note that the following book-marked content (selected from DirectMusic Producer's Bookmarks Toolbar) has aspects that will be implemented differently or will not be supported on Xbox at launch.
