April 2001 XDK Update Instructions

1. Summary

This release updates the software on your target system and the Software Development Kit (SDK) on your development system. To update:

  1. Install the updated SDK on your development system per section 4.1.
  2. Update your target system software per section 5.1.
  3. If necessary, establish network communication between the two systems per section 6.
  4. If you wish to use a TV display with your target system, change your target system display settings per section 7.

2. April 2001 XDK Update Contents

This update to the Xbox Development Kit (XDK) for the Xbox™ video game system from Microsoft consists of the following components:

  • Xbox SDK disc—contains the latest Xbox SDK and other tools to be installed on your development system.
  • Xbox target system recovery disc—use the recovery disc to upgrade the XDK target system to the latest version of the Xbox System Software.

    • 4.   Xbox Development System Setup

    The development system is the computer that runs the Microsoft® Visual C++® compiler and is where the Xbox SDK must be installed. Installing the latest version of the SDK is the only step required on the development system for the April 2001 XDK release, as long as the development system meets the requirements for the February 2001 release (Microsoft® Windows® 2000 with Service Pack 1 and Visual C++ with Service Pack 4 and the Processor Pack).

    Note   Code written to run on the Xbox System Software must be run and tested only on the XDK target system provided by Microsoft. Do not run code written for Xbox on a development system or any system other than the target system.

    4.1   SDK Installation

    Before you can install the XDK software on Windows 2000, you must have permissions at the level of Administrator. Refer to the Windows help system for further information.

    It is not necessary to uninstall a prior release of the Xbox SDK before installing this update. If you do not have a prior release of the Xbox SDK installed, first perform the steps in section 4.2 below.

    Install the Xbox SDK from the April 2001 SDK disc. To install, insert the disc into your development system CD drive, select Install Xbox SDK - April 2001 from the installation screen, and follow the instructions.

    Note   When the installation program copies an updated file to the SDK directory on your hard disk, it will overwrite the existing file even if that file has been modified by you since the last SDK install. For example, if you have made changes to the sample or tutorial code installed with a previous SDK, but left those modified files in the same location and with the same filename, the update will overwrite those files with new versions and you will lose any changes you might have made. Copy any modified files you wish to retain to a new location prior to installing the SDK.

    4.2   No Prior SDK Installed

    If you do not have a prior release of the Xbox SDK installed, follow these instructions before downloading and running the SDK update:

    1. Your development system must be running Windows 2000 Service Pack 1. If you have not already done so, upgrade your development system to Windows 2000 with Service Pack 1 (SP1 is downloadable from www.microsoft.com/windows2000/downloads/recommended/sp1/default.asp). For installation instructions, see the Windows 2000 Service Pack 1 documentation. Windows 2000 with Service Pack 1 is the only operating system supported by the Xbox Development Kit.

    2. Install Microsoft Visual C++ 6.0 Professional. For installation instructions, see the Visual C++ documentation. This step is not necessary if a licensed copy of Visual C++ 6.0 is already installed on the development system.

    3. Install Visual C++ 6.0 Service Pack 4 from the Service Pack 4 disc included with the XDK. The install program runs automatically when the disc is inserted into the development system CD drive. To install, follow the instructions on the screen. It is not necessary to install Visual C++ Service Pack 3.

      Note   Service Pack 4 does not include an uninstall utility. If you want to uninstall Service Pack 4, you must uninstall and reinstall Visual C++ 6.0.

    4. Install the final version of the Microsoft Visual C++ Processor Pack from the April 2001 SDK disc. To install, insert the disc into your development system CD drive, select Install Visual C++ Processor Pack from the installation screen, and follow the instructions (the Processor Pack is located in the VCPP directory on the disc). If an earlier version of the Processor Pack is installed, setup will fail with an error that is the same as the error you would receive if you did not have Windows 2000 Service Pack 1, Visual C++ Service Pack 4, or the Visual C++ Processor Pack installed.

    5.   Xbox Target System Setup

    The target system is the silver computer that came with the XDK; it is configured to represent the Xbox video game console.

    5.1   Installing the Updated Xbox System Software on the Target System

    Installing the latest version of Xbox System Software is the only step required on the target system for the April 2001 XDK release, provided you have already properly configured the target system hardware you received with earlier versions of the XDK.

    The April 2001 release of the XDK includes an updated target system recovery disc that contains a new version of the Xbox System Software. To update your target system to the Xbox System Software, boot your target system from the updated target system recovery disc. You will also use this same process if you need to restore the target system hard disk due to corruption.

    To install or recover the updated Xbox System Software:

    1. Insert the updated target system recovery disc in the target system disc drive and reboot the system.
    2. You will be prompted to press any controller button.
    3. After the target system is updated, the disc drive tray will open, and you will be prompted to remove the CD and press any controller button to reboot.
    4. The Title Launcher screen will be displayed in the Settings: Machine Name mode. Enter a name for your target system for network use; select OK when you are finished.

    WARNING:   Booting from the target system recovery disc will erase the data on the target system hard disk. Copy any data that you wish to retain to another system or storage device before booting from the target system recovery disc.

    Note   You must connect a VGA monitor to the target system while performing the recovery update. After the update is complete, you can reconnect a TV monitor. See section 7 below.

    Note   If you downloaded the April 2001 XDK Update from the beta support Web site (http://www.betaplace.com) rather than receiving an actual disc from Microsoft, you must burn your own April 2001 XDK recovery disc image using the April_2001.iso file that is available for download on the beta support Web site. For more information about burning a new recovery disc image using April_2001.iso, see the documentation for your CD image burning software (Adaptec Easy CD Creator or similar software).

    6.   Establish a Network Connection Between Target and Development Systems

    If you changed the machine name of the Xbox target system, or you are installing the SDK for the first time on this development system, you must set up the network connection between the target and development systems. After completing all the steps in Xbox Development System Setup and Xbox Target System Setup, and with both systems running:

    1. From the development system, click the Start button, click Run, and type "cmd" in the Open: window that appears.
    2. From the C:\ prompt in the cmd.exe window, type "xbdir -x name xe:\", where name is the name of your target system entered during the Installing the Updated Xbox System Software on the Target System step. In place of a name that you assign, you may enter the IP address of the target system (the IP address is displayed at the bottom of the Title Launcher, for example, (123.45.67.89)).

    Note   If the target system and the development system are not on the same subnet, you must enter the IP address instead of the target system name.

    Note   If steps 1 and 2 are not accomplished and you attempt to build an Xbox executable, a No address for Xbox has been set compiler error message will result.

    Note   You can use the "xbdir xe:\" command at any time to verify that the two systems are communicating.

    7.   Connecting a TV

    To create the most accurate run-time environment for testing Xbox games, it is recommended that a TV be used for the Xbox target system monitor. After setting up both systems (sections 4–6), you may connect a TV. To change the target system display settings, press the X controller button (A on the controllers originally distributed with the XDK) from the Title Launcher screen. You will need to specify the type of television (NTSC or PAL) and the type of cable you are using (S-video or S-video/composite converter, neither of which is included with the XDK). The display settings will not be updated until the target system is rebooted.

    Note   If you have changed the display settings such that you can no longer see video from the target system, you can open a command window on the development system and type "xbsetcfg -v MONITOR" to reset the display settings so that the target system will work when connected to a VGA monitor. You will need to reboot the target system after running the "xbsetcfg" command before the display settings change will take effect.

    8.   To Uninstall the Xbox SDK

    1. Launch the Add/Remove Programs dialog from the Windows Control Panel, select Microsoft Xbox SDK, and then click Change/Remove.
    2. Setup will detect the previous installation of the Xbox SDK. Follow the instructions on the screen.

    Note   Xbox SDK uninstallation may not delete all files installed with the Xbox SDK. To complete uninstallation in this case, manually delete the Xbox SDK root folder (located at C:\Program Files\Microsoft Xbox SDK, by default).

    For uninstallation of Visual C++, see the Visual C++ documentation. You cannot uninstall Visual C++ Service Pack 4 or the Processor Pack separately.
